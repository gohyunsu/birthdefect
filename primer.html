0 · vocabulary

First, separate the things that are often confused.

모델은 원본 image를 직접 “이해”하지 않습니다. image, feature, prediction, label, loss는 서로 다른 단계의 산출물입니다. 이 다섯 가지를 구분하면 두 논문의 수식이 훨씬 단순해집니다.

Input

Image / frame

초음파 동영상의 한 장면처럼 센서가 기록한 원시 관측입니다. pixel 값의 격자이며, 아직 “심장 구조”라는 의미를 갖지 않습니다.

Representation

Feature / embedding

신경망이 image 또는 node를 숫자 벡터로 바꾼 결과입니다. 비슷한 관측이 가까워지도록 학습되면, 거리나 내적으로 유사도를 계산할 수 있습니다.

Output

Prediction / score

class probability, bounding box, anomaly score처럼 실제 과제에 쓰는 값입니다. SAD의 score는 “이상일 확률”이라기보다 정상 reference에서 얼마나 벗어났는지 판단하기 전의 raw scalar입니다.

Supervision

Label / loss

label은 사람이 제공한 정답이고, loss는 prediction과 원하는 상태의 차이를 수치화한 학습 신호입니다. optimizer는 loss가 작아지도록 parameter를 갱신합니다.

1 · image features

A detector is a sequence of narrowing questions.

SELSA는 Faster R-CNN detector 위에 올라갑니다. 따라서 먼저 “이 frame에서 무엇이 어디에 있나?”라는 detector의 기본 문제를 풀고, 그 다음 “다른 frame의 어떤 후보가 이 판단을 도울까?”를 묻습니다.

Backbone은 image를 feature map으로 바꾸고, RPN은 object가 있을 법한 box 후보(proposal)와 objectness를 냅니다. 각 proposal의 ROI feature가 SELSA가 비교·집계하는 단위입니다. Faster R-CNN의 RPN은 shared convolutional feature 위에서 후보 위치와 objectness를 end-to-end로 예측합니다.

Concrete example. four-chamber view에서 RPN이 심장 주변 후보 box 300개를 만들 수 있습니다. 이 중 한 proposal의 ROI feature가 x입니다. SELSA는 같은 clip의 다른 frame에서도 x와 의미적으로 닮은 proposal feature를 찾아 보강합니다. 즉, pixel 전체가 아니라 “후보 구조를 나타내는 벡터”끼리 비교합니다.

2 · similarity and attention

Attention is a weighted read, not magic.

Attention의 기본 동작은 간단합니다. 현재 알고 싶은 대상인 query가 있고, 참고할 candidate마다 유사도 score를 계산합니다. score를 softmax로 0~1 가중치로 바꾼 뒤, candidate feature를 가중 평균합니다.

SELSA의 φ(xᵢᵏ)ᵀψ(xⱼˡ)는 바로 이 score입니다. φ와 ψ는 같은 feature를 각각 query/key 역할에 맞게 변환하는 학습 함수입니다. 이는 query-key-value attention의 간결한 형태로 이해할 수 있습니다.

Queryunclear target frame
compares with
candidate Ascore 2.0weight 0.65
candidate Bscore 1.0weight 0.24
candidate Cscore 0.2weight 0.11
reads
Aggregated feature0.65A + 0.24B + 0.11C

Why softmax?

raw score는 음수나 큰 수가 될 수 있어 바로 평균의 비중으로 쓰기 어렵습니다. softmax는 모든 candidate weight를 양수로 만들고 합을 1로 맞춥니다. 그래서 “어느 후보를 얼마나 읽을지”가 해석 가능한 비율이 됩니다.

αⱼ = exp(scoreⱼ) / Σₘ exp(scoreₘ)

What can go wrong?

유사도가 anatomy가 아니라 background, device overlay, probe angle 같은 shortcut을 잡으면 잘못된 candidate가 큰 weight를 받습니다. 따라서 attention weight만으로 임상 근거를 주장할 수 없고, hard case와 subgroup에서 실제로 무엇을 참조하는지 감사해야 합니다.

x̄ = Σⱼ αⱼ · xⱼ

3 · graph and time

A graph makes relationships explicit.

Graph는 data type이 아니라 표현 방식입니다. node는 관측 단위, edge는 두 관측이 관련 있다는 주장입니다. GNN은 “내 feature만 보지 말고 연결된 이웃의 정보를 모아 representation을 갱신하라”는 계산입니다.

SAD는 continuous-time interaction graph를 다룹니다. event δ(t) = (vᵢ, vⱼ, t, xᵢⱼ)는 i와 j가 시각 t에 interaction했고 edge feature가 xᵢⱼ라는 뜻입니다. TGAT는 이웃 정보에 상대 시간차를 함께 넣어, 같은 이웃이라도 얼마나 오래전 관계인지 다르게 처리합니다.

frame t−2frame t−1target at tsimilar view
aggregate messagesneighbor feature + edge feature + relative timeupdated embedding zᵢ(t)

Static vs dynamic graph

static graph에서는 연결이 고정된 한 장의 지도입니다. dynamic graph에서는 event가 순서대로 들어오고, node/edge attribute와 관계가 바뀝니다. 그래서 train/test split도 time order를 지켜야 미래 정보가 과거 representation에 새지 않습니다.

Why use TGAT in SAD?

SAD는 temporal encoder 자체를 새로 제안한 논문이 아니라, TGAT를 backbone으로 사용합니다. SAD의 새 기여는 그 representation 위에 time-equipped memory와 score-derived contrastive learning을 결합해 희소 라벨을 더 활용하는 데 있습니다.

4 · learning with few labels

Three jobs that must not be mixed up.

SAD에서 labeled data와 unlabeled data는 같은 방식으로 쓰이지 않습니다. label은 “정답 방향”을 주고, memory bank는 “최근 정상 기준선”을 만들며, pseudo-label contrastive learning은 “representation을 정돈”합니다. 세 역할을 구분해야 SAD의 의도를 오해하지 않습니다.

A · labeled samples

Deviation loss

labeled normal의 deviation은 0에 가깝게, labeled anomaly는 margin보다 멀어지게 합니다. score의 절대값이 아니라 현재 reference mean·standard deviation에서의 z-score가 중요합니다.

dev = (s − μᵣ) / σᵣ
+
B · mostly-normal history

Memory bank

normal 또는 unlabeled score와 time을 FIFO queue에 저장합니다. 최근 항목에 큰 weight를 줘 distribution drift를 반영한 μᵣ, σᵣ를 추정합니다.

recent score → higher weight
+
C · unlabeled samples

Contrastive loss

deviation 차이가 작은 node끼리는 embedding을 가깝게 합니다. 그러나 이것은 “의사 정답”이므로 threshold·warm-up·오류 감사를 반드시 동반해야 합니다.

close dev → positive pair
Important distinction. pseudo-label은 새 ground truth가 아닙니다. 모델 자신의 현재 추정으로 만든 임시 학습 신호입니다. 초기 score가 체계적으로 틀리면 error reinforcement가 생길 수 있으므로, SAD를 의료 데이터에 적용한다면 calibration, confidence filtering, label-ratio ablation, clinician review가 필수입니다.
